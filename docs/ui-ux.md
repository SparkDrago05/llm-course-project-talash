# UI/UX Design Thinking Notes (Milestone 1)

## Design Goals
1. Minimize recruiter effort with a single upload-to-summary flow.
2. Surface uncertainty quickly using completeness metrics and flags.
3. Keep candidate details traceable to extracted fields.

## Screen Intent
1. Upload View
- Primary action is CV ingestion.
- Secondary action opens processing queue or status.

2. Candidate Summary Dashboard
- Global health view for batch processing.
- Highlights completeness and top alerts for quick triage.

3. Candidate Detail View
- Single-candidate snapshot with extracted sections.
- Missing-info flags and email draft preview to support follow-up.

## Milestone 1 Constraints
- Wireframes are low-fidelity to validate flow and information hierarchy.
- Interactive frontend implementation is deferred to next milestone.
